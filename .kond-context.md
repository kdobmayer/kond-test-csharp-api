# Context Bundle: DocManager Sharing Feature

## Build and Test Commands
- **Build:** `dotnet build`

## Project Overview
- **Framework:** ASP.NET Core 8.0
- **Database:** SQLite with EF Core (InMemory for tests)
- **Architecture:** Controller -> Service/DbContext (No separate repository layer currently)
- **Mapping:** Manual mapping in `MappingService.cs`
- **Authorization:** Manual checks in controller actions using `requestingUserId`

### Manual Authorization (DocumentsController.cs)
```csharp
[HttpDelete("{id}")]
public async Task<ActionResult> Delete(int id, [FromQuery] int requestingUserId)
{
    var doc = await _db.Documents.FindAsync(id);
    if (doc == null)
        return NotFound(new { message = "Document not found" });

    var user = await _db.Users.FindAsync(requestingUserId);
    if (user == null || !user.IsActive)
        return Forbid();

    if (doc.CreatedByUserId != requestingUserId)
        return Forbid();
    // ...
}
```

## Constraints and Gotchas
- **No AutoMapper:** Use `MappingService.cs`.
- **Manual Auth:** You must explicitly check if the user is active and if they own the document OR have a valid share.
- **Timestamps:** Use `DateTime.UtcNow`.
- **Cascade Deletes:** Ensure `DocumentShare` records are deleted when the associated `Document` is deleted.
- **Repository Pattern:** The prompt mentions repository methods, but the project currently uses `AppDbContext` directly in controllers and services. You should decide whether to introduce repositories or implement the "repository methods" in a way that fits the project.
- **SQLite/InMemory:** Be aware of EF Core provider differences, though standard LINQ should work fine.
